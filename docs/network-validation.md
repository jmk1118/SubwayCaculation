# Network Validation Guide

이 문서는 지하철 노선 JSON 데이터 점검을 수동으로 수행할 때 기준으로 사용한다.

## 목적
- `public/data/*.json`의 그래프 연결 무결성 확인
- 잘못된 환승 연결 또는 누락 연결 발견 시 수정 기준 통일

## 기본 점검 항목
1. `neighbors`에 존재하지 않는 노드 ID가 없어야 한다.
2. 양방향 연결이어야 한다.
- `A -> B`가 있으면 `B -> A`도 있어야 한다.
3. 노선 파일 내부 선형 연결이 끊기지 않아야 한다.
4. 환승역은 동일 역명 노드끼리만 연결한다.
5. 노선명, id suffix, 파일명이 일관되어야 한다.

## 수동 점검 절차
1. 데이터 수정 후 무결성 검사 실행
- `npm run validate:seoul-lines` (현재 등록된 검증 스크립트)
2. 전체 빌드 확인
- `npm run build`
3. 결과 확인
- 실패 시 누락/비대칭/단절 원인 수정

## 노선 추가/수정 원칙
1. 신규 노선 파일 생성 후 기존 포맷 유지
- `{ id, name, line, neighbors }`
2. 선로 연결은 실제 노선 순서 기준
3. 환승 연결은 동일 역명 기반 양방향 추가
4. 환승 추가 후 전체 무결성 재검증

## 실제 노선도 대조 원칙
1. 공식 기관(운영사/공공데이터/공식 홈페이지) 우선
2. 공식 자료가 불명확할 때만 보조 출처 참고
3. 확인된 변경만 반영하고 추정 수정 금지

## 작업 기록 권장 포맷
- 변경 파일 목록
- 추가/수정 역 목록
- 환승 연결 변경 목록
- 검증 결과(`missing`, `asymmetric`, `build`)
